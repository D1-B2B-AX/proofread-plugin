# 교안 검수 플러그인 — OM 사용 가이드

> 교안(PPT/PDF/노션)의 오탈자, 맞춤법, 고객사명 불일치 등을 자동으로 검수하는 Claude Code 플러그인입니다.

---

## 1. 사전 준비 (최초 1회)

### 1-1. Claude Code 설치 확인

Claude Code가 설치되어 있어야 합니다. 터미널(명령 프롬프트)에서 아래 명령어로 확인:

```
claude --version
```

버전이 표시되면 설치 완료 상태입니다.

### 1-2. 플러그인 설치

Claude Code 대화창에서 아래 명령어를 입력하면 자동 설치됩니다:

```
git clone https://github.com/D1-B2B-AX/proofread-plugin.git ~/.claude/commands/proofread-plugin
```

첫 실행 시 필요한 Python 패키지(python-pptx, PyPDF2 등)가 자동으로 설치됩니다.

### 1-3. 권한 설정 (중요!)

**이 설정을 하면 검수 중 Enter를 여러 번 누를 필요 없이, 자동으로 쭉 진행됩니다.**

아래 방법 중 하나를 선택하세요:

**방법 A: cc 명령어로 실행 (★ 추천)**
터미널에서 `claude` 대신 `cc`를 입력하면 권한 승인 팝업 없이 바로 실행됩니다.
시작 시 1번만 "Yes, I accept"를 선택하면 이후 끊김 없이 작업 가능합니다.

**방법 B: 세션 시작 시 설정 (매번)**
1. Claude Code 실행
2. 처음 도구 실행 승인 팝업이 뜨면 **"Allow for this session"** 선택
3. 이후 해당 세션에서는 자동 실행

**방법 C: 영구 설정 (1회만)**
1. Claude Code 실행
2. `/permissions` 입력
3. `Bash("python3*")` → Allow 추가
4. `Write` → Allow 추가
5. 이후 매번 자동 실행

> **권한 설정을 안 하면?**
> 텍스트 추출 스크립트 실행, 결과 파일 저장 등 매 단계마다 "실행해도 되나요?" 팝업이 뜨고, Enter를 눌러야 진행됩니다. 검수 자체에는 문제 없지만 번거로울 수 있습니다.

## 2. 검수 실행 방법

### 2-1. 기본 실행

1. Claude Code 실행
2. `/교안검수` 입력 후 Enter
3. 아래 4가지 정보를 안내에 따라 입력:

```
1. 교안 파일 경로 (예: 바탕화면/교안.pptx 또는 드래그)
2. 고객사명 (예: KT, 삼성전자)
3. 과정명 (예: AI 활용 실무)
4. 강사명 (예: 김철수)
```

4. Enter → 자동으로 텍스트 추출 → 검수 → 결과 출력

### 2-2. 파일 경로 쉽게 입력하기

파일 경로를 직접 타이핑하지 않아도 됩니다:

- **Windows**: 파일을 **Shift + 우클릭** → "경로로 복사" → 붙여넣기
- **드래그 앤 드롭**: 일부 터미널에서는 파일을 터미널 창에 드래그하면 경로가 자동 입력됩니다

### 2-3. 지원 파일 형식

| 형식 | 확장자 | 비고 |
|------|--------|------|
| PowerPoint | `.pptx` | 가장 정확한 추출 |
| PDF | `.pdf` | 이미지 위주 페이지가 30% 이상이면 OCR로 텍스트 판독 시도 |
| 노션 | 직접 붙여넣기 또는 PDF 저장 | 아래 3번 참고 |

---

## 3. 노션 교안 검수 방법

노션으로 작성된 교안은 아래 방법 중 하나를 사용하세요:

### 방법 1: 텍스트 복사-붙여넣기 (★ 추천)

**가장 정확한 검수가 가능합니다.** 띄어쓰기가 정상 유지되어 오탈자/띄어쓰기 모두 검수 가능.

1. 노션 페이지에서 `Ctrl+A` (전체 선택) → `Ctrl+C` (복사)
2. `/교안검수` 실행 후, 입력 방식에서 "텍스트 붙여넣기" 선택
3. 복사한 내용 붙여넣기

### 방법 2: PDF로 저장 후 파일 경로 입력

1. 노션 페이지를 **브라우저**에서 열기
2. 브라우저 우측 상단 점 3개(⋮) 클릭 → **"인쇄"** 선택
   - ⚠️ `Ctrl+P`는 노션이 검색창으로 가로채므로, 반드시 브라우저 메뉴에서 "인쇄" 클릭
3. 프린터를 **"Microsoft Print to PDF"**로 변경
4. ⚠️ **"머리글 및 바닥글" 체크 해제** (URL이 불필요하게 포함되는 것 방지)
5. "인쇄" 클릭 → 파일 저장
6. 저장된 PDF 파일 경로를 입력

> ⚠️ **참고**: PDF 변환 시 띄어쓰기가 누락될 수 있습니다. 오탈자 검수는 가능하지만, 띄어쓰기 검수 정확도는 방법 1보다 낮습니다.
> 💡 **토글이 있는 노션 교안**: PDF 저장 전에 토글을 모두 펼친 상태에서 저장해야 토글 안의 내용도 포함됩니다.

### 방법 3: 강사에게 마크다운(.md) 내보내기 요청 (위 방법이 안 될 때)

강사에게 노션 페이지를 마크다운(.md)으로 내보내기 요청. 띄어쓰기가 정상 유지됩니다.

## 4. 결과 확인 및 활용

### 4-1. 결과 구조

검수 결과는 **3개 등급**으로 분류되어 출력됩니다:

| 등급 | 의미 | 조치 |
|------|------|------|
| **[오류 확정]** | 명백한 오탈자, 맞춤법 오류 | 강사에게 수정 요청 |
| **[OM 확인 필요]** | 의도적일 수 있어 판단 필요 | OM이 확인 후 결정 |
| **[참고사항]** | 틀리진 않지만 알아두면 좋은 것 | 강사 전달 불필요, OM 참고용 |

### 4-2. 결과 파일 위치

검수 결과는 대화창에 표시됨과 동시에 HTML 파일로 자동 저장됩니다:

```
바탕화면/claude_temp/검수결과_[파일명]_[날짜].html
```

### 4-3. 강사 수정 요청 메일

**[오류 확정]**이 1건 이상이면 강사 수정 요청 메일 초안이 자동 생성됩니다.
- 발신자명은 `OOO`으로 비워져 있으니, 본인 이름으로 변경 후 사용하세요
- **[참고사항]** 항목은 메일에 포함되지 않습니다 (내부 참고용)

## 5. 2차 검수 (수정본 확인)

강사가 수정본을 보내오면, 수정 사항이 제대로 반영되었는지 확인할 수 있습니다.

1. `/교안검수` 실행
2. 안내에서 **"2차 검수"** 입력
3. 수정된 교안 파일 경로 입력
4. 1차 검수 결과와 비교하여 수정 완료/미수정/다르게 수정 여부를 표로 출력

## 6. 예상 소요시간

| 분량 | 예상 시간 | 비고 |
|------|----------|------|
| 50p 이하 | 2~3분 | |
| 50~100p | 3~5분 | |
| 100~200p | 5~10분 | |
| 200p 이상 | 10~20분 | |

**소요시간에 영향을 주는 요인:**
- 한글 서술 비중이 높을수록 오래 걸림 (영문/코드 위주 교안은 빠름)
- 오류가 많은 교안일수록 분석·분류에 시간 소요
- 표가 많고 복잡할수록 처리 시간 증가
- 이미지 위주 페이지가 많으면 오히려 빠름 (OCR 미실행 시 검수 대상에서 제외)

## 참고: 전체 검수 항목

| 검수 항목 | 설명 |
|----------|------|
| 오탈자 | 한글 맞춤법 오류, 잘못된 조사, 누락/중복 글자 |
| 의심 단어 | 형태소 분석기가 사전에 없는 단어로 감지한 항목 |
| 영문 스펠링 | 영문 사전에 없는 단어 (고유명사/브랜드명 제외) |
| 반복 변형 | 같은 텍스트가 반복되는데 1글자만 다른 경우 |
| 텍스트 깨짐 | □, ?, 자모 분리 등 폰트/인코딩 문제 |
| 띄어쓰기 | 한글 맞춤법에 맞지 않는 띄어쓰기 |
| 전각문자 | 의도치 않은 전각 괄호, 전각 숫자 등 |
| 고객사/과정명 | 입력한 고객사명·과정명과 교안 내 표기 불일치 |
| 강사명 | 다른 강사명 잔존 (이전 교안 재활용 시) |
| 개인정보 | 전화번호, 이메일, 주민번호 등 민감정보 노출 |
| 문맥 검수 | 주술 호응, 용어 일관성, 문장 완결성, 앞뒤 모순 |

## 7. 자주 묻는 질문

**Q. PDF에서 띄어쓰기가 이상하게 나와요.**
→ PDF 변환 과정에서 띄어쓰기가 누락되는 것은 원본 PDF의 특성입니다. 플러그인이 PDF 추출 한계로 판단되는 건은 자동으로 걸러냅니다. 명백한 오류만 결과에 포함됩니다.

**Q. 이미지로 된 페이지는 검수가 안 되나요?**
→ 이미지 위주 페이지가 전체의 30% 이상이면 OCR(이미지 속 텍스트 판독)을 자동 실행합니다. 다만 OCR로 발견된 항목은 오독 가능성이 있어 "확인 필요"로 분류됩니다. 이미지 위주 페이지가 30% 미만이면 해당 페이지는 검수 대상에서 제외되며, 결과에 "이미지 위주 페이지 N개"로 안내됩니다.

**Q. PPT가 `.ppt` 형식이에요.**
→ `.ppt`(구 형식)는 지원하지 않습니다. 강사에게 `.pptx`로 저장하여 다시 보내달라고 요청해 주세요. (PowerPoint에서 "다른 이름으로 저장" → `.pptx` 선택)

**Q. 오류가 없다고 나왔는데 정말 없는 건가요?**
→ 텍스트 기반 검수이므로 이미지 속 텍스트, 디자인 요소 등은 확인할 수 없습니다. 텍스트로 추출 가능한 내용에 한해 검수가 이루어집니다.

**Q. 검수 결과를 노션에 올릴 수 있나요?**
→ 네, 검수 완료 후 "노션에 업로드해줘"라고 말하면 결과 파일이 노션에 자동 업로드됩니다.
