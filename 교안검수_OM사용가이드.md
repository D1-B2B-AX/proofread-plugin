# 교안 검수 플러그인 — OM 사용 가이드

## 🧐 이 플러그인은 뭔가요?

교안 파일(PPT, PDF)을 넣으면 오탈자, 맞춤법, 영문 스펠링, 텍스트 깨짐 등을 자동으로 찾아주는 도구예요.

검수 결과는 브라우저에서 보기 좋은 HTML 페이지로 나오고, 강사에게 보낼 메일 초안까지 자동으로 만들어 줍니다.

---

## 설치 방법

> Claude Code 대화창에서 아래 명령어를 입력하면 자동 설치됩니다:
>
> ```
> git clone https://github.com/D1-B2B-AX/proofread-plugin.git ~/.claude/commands/proofread-plugin
> ```
>
> 처음 실행 시 필요한 패키지가 자동으로 설치됩니다.

---

## 📋 사용법 (3단계)

### 🔹STEP 1. 플러그인 실행

클로드 코드 터미널에서 아래와 같이 입력해요:

```
/교안검수
```

그러면 이런 안내가 나옵니다:

```
교안 검수를 시작합니다.

교안 파일을 대화창에 드래그해 주세요. (PPT, PDF 지원)
또는 파일 경로를 직접 입력해도 됩니다.

💡 PPT와 PDF 둘 다 있다면 PPT가 더 정확해요.
💡 노션 교안이면 "노션"이라고 입력해 주세요.
💡 수정본 확인은 "2차 검수"라고 말씀해 주세요.
```

### 🔹STEP 2. 교안 파일 넣기

**방법 1)** 교안 파일을 대화창에 드래그 앤 드롭

**방법 2)** 파일 경로를 직접 입력 (예: C:\Users\...\교안.pptx)

파일을 넣으면 고객사/과정명/강사명을 자동으로 파악해서 보여줍니다:

```
파일: 강의교재_AI활용_2025.07.03.pptx (120p)

아래 정보가 맞는지 확인해 주세요:
- 고객사: HL그룹
- 과정명: AI 활용 실무
- 강사명: 이지훈

맞으면 "확인", 수정할 내용이 있으면 알려주세요.
```

맞으면 "확인" 이라고 입력하면 바로 검수가 시작돼요.

> **🔸PPT와 PDF 둘 다 있다면?**
> PPT가 더 정확해요. PPT 원본이 있으면 반드시 PPT로 검수하세요.

> **🔸노션 교안인 경우?**
> "노션"이라고 입력하면 노션 교안 입력 방법을 안내해 드려요.
>
> 💡추천 방법: 노션 페이지를 브라우저에서 열고 → Ctrl+P → "Microsoft Print to PDF"로 저장
>
> 다만 토글 안의 내용도 검수하려면, PDF 저장 전에 토글을 모두 펼친 상태에서 저장해야 해요.
> 텍스트 복사-붙여넣기로 할 경우에는, 토글을 모두 펼친 뒤 드래그로 전체 선택 → 복사(Ctrl+C) → 대화창에 붙여넣기하면 PDF 변환 없이 바로 검수할 수 있어요.
> (단, 이 경우 페이지 번호 없이 결과가 나옵니다)

> **🔸더 빠르게 시작하려면?**
> /교안검수 입력할 때 파일을 함께 드래그하면, 안내 과정 없이 바로 검수가 시작돼요.

### 🔹STEP 3. 결과 확인

검수가 끝나면 브라우저가 자동으로 열리면서 결과 페이지가 표시돼요.

---

## 🔎 결과 페이지 보는 법

결과는 4가지로 분류됩니다:

| 분류 | 색상 | 의미 | 어떻게 하면 되나요? |
|------|------|------|---------------------|
| **오류 확정** | 🔴 빨강 | 누가 봐도 틀린 것 | 강사에게 수정 요청 |
| **OM 확인 필요(상)** | 🟠 주황 | 코드/주석 불일치 등 교안 내용 확인 필요 | 강사에게 확인 요청 |
| **OM 확인 필요** | 🟠 주황 | 의도일 수도 있는 것 | OM이 직접 판단 후 결정 |
| **참고사항** | ⚪ 회색 | 틀리진 않지만 알아두면 좋은 것 | 참고만 (강사에게 전달 불필요) |

[OM 확인 필요] 안에서 심각도 "상"이 표시된 항목(코드/주석 불일치, 코드 오류, 수식 불일치 등)은 교안 내용의 정확성 확인이 필요한 항목입니다. 이 항목은 OM이 직접 판단하기 어려우므로 메일 초안에 자동 포함되어 강사에게 확인을 요청합니다.

### 오류 확정의 심각도 / 확인 필요의 심각도

| 심각도 | 의미 |
|--------|------|
| **상** | 고객 눈에 바로 보이는 오류 (제목, 상단 등) |
| **중** | 주의 깊게 보면 발견되는 오류 |
| **하** | 사소한 표기 오류 |

확인 필요의 심각도는 "상"만 있습니다. 코드/주석 불일치, 코드 오류, 수식 불일치 등 교안 내용 확인이 필요한 항목이 해당되며, 이 항목은 메일 초안에 자동 포함됩니다. 심각도가 없는(-) 항목은 OM이 직접 판단하는 일반 확인 항목입니다.

### 🔹STEP 4. 수정본 재검수 (필요 시 활용)

강사가 수정본을 보내오면, 1차에서 발견된 오류가 제대로 반영됐는지 확인할 수 있어요.

1. /교안검수 실행
2. "2차 검수" 라고 입력
3. 수정된 교안 파일을 드래그
4. 1차 결과와 비교하여 수정 완료 / 미수정 / 다르게 수정 여부를 표로 보여줌

---

## 📧 메일 초안 사용법

오류 확정이 1건 이상이거나 확인 필요(상)이 1건 이상이면 결과 페이지 하단에 메일 초안이 자동으로 생성돼요.

1. "메일 복사" 버튼 클릭 → 제목+본문이 클립보드에 복사됨
2. 메일 제목란에 첫 줄 [제목] 내용을 붙여넣기, 본문란에 나머지 내용 붙여넣기
3. 필요 시 내용 수정 후 발송

> 확인 필요(상)은 코드/주석 불일치 등 교안 내용 확인 사항으로, 메일에 자동 포함됩니다. 확인 필요 중 심각도가 없는(-) 항목과 참고사항은 메일에 포함되지 않아요. OM 내부 참고용이에요.

## 💻 어떤 오류를 잡아주나요?

### 자동으로 잡아주는 것

- 한글 오탈자 (예: 시서템→시스템, 몸무계→몸무게)
- 영문 스펠링 오류 (예: Trafic→Traffic, Evants→Events)
- 텍스트 깨짐 (□□□, ??? 등 폰트 문제로 깨진 글자)
- 반복 텍스트 변형 (같은 문장이 여러 번 나오는데 1글자만 다른 경우)
- 띄어쓰기 오류 (예: 할수있다→할 수 있다)
- 전각문자 감지 (의도치 않은 전각 괄호, 전각 숫자 등)

### AI가 문맥을 보고 잡아주는 것

- 주술 호응 오류
- 앞뒤 페이지 내용 모순 (예: "3단계" → "4단계")
- 문맥에 맞지 않는 유사 단어 (예: "교생" → "교실")
- 순서/단계 패턴 대조 (예: TOFU → MOFU → BOFU 순서에서 BOFU 자리에 TOFU가 적혀있으면 확인 필요로 플래그)
- 용어 표기 불일치 (예: "워크플로우" vs "워크플로" 혼용)
- 다른 고객사명/강사명 잔존 (이전 교안 재활용 시)
- 개인정보 노출 (전화번호, 이메일 등)

### 오류로 잡지 않는 것

- 외래어 표기 차이 (컨텐츠/콘텐츠, 멤버쉽/멤버십 등) → 참고사항으로만 안내
- 전문 용어, 고유명사, 브랜드명
- 코드, URL, 이메일(강사 연락처로 의도된 것)
- 슬라이드 제목의 명사형 종결 (교안 특성상 정상)

## ❓ 자주 묻는 질문

**Q. 검수에 얼마나 걸리나요?**
- 50p 이하: 3~5분
- 50~150p: 5~10분
- 150p 이상: 10~20분

**Q. PDF 교안인데 띄어쓰기가 이상하게 나와요.**

PDF는 변환 과정에서 띄어쓰기가 누락될 수 있어요. 플러그인이 이 점을 감안해서, PDF에서는 명백한 오류만 잡고 애매한 건 넘어갑니다.

**Q. 노션 교안은 어떻게 넣나요?**

노션 페이지를 브라우저에서 열고 → 토글이 있다면 모두 펼친 후 → Ctrl+P → "Microsoft Print to PDF"로 저장 → 저장된 PDF를 드래그하면 돼요.

**Q. 수정본을 다시 검수하고 싶어요.**

/교안검수 실행 후 "2차 검수"라고 입력하면, 1차 결과와 비교해서 수정 반영 여부를 확인해 줘요.

**Q. 이미지 안의 텍스트도 검수되나요?**

PPT에서 텍스트 박스로 작성된 내용은 모두 검수돼요. 이미지 위주 페이지가 전체의 30% 이상이면 OCR(이미지 속 텍스트 판독)을 자동 실행합니다. 다만 OCR로 발견된 항목은 오독 가능성이 있어 "확인 필요"로 분류됩니다. 30% 미만이면 해당 페이지는 검수 대상에서 제외되며, 결과에 "이미지 위주 페이지 N개"로 안내됩니다.

## ‼️ 플러그인 한계점

### A. 구조적으로 잡을 수 없는 것 (한계)

이 항목들은 플러그인의 구조적 특성상 감지가 불가능하거나, 감지해도 신뢰할 수 없는 영역입니다.

**A-1. 이미지 안의 텍스트 (텍스트 레이어 없음)**
- PPT에서 이미지로 삽입된 텍스트는 추출 자체가 불가능
- 이미지 위주 페이지가 30% 이상일 때만 OCR 시도하며, OCR 결과는 모두 "확인 필요"로 분류

**A-2. 내용(지식)의 정확성 검증**
- "이 설명이 사실인가?"는 판단 불가 (예: "Python은 1995년에 만들어졌다" → 실제 1991년이지만, 플러그인은 판단 불가)
- 코드/주석 불일치는 감지 가능하나, 코드 로직의 정확성 자체는 보장하지 않음

**A-3. LLM 판단의 비결정성 (매번 결과가 다를 수 있음)**
- AI 기반 검수 항목(문맥 검수, 유사 단어 등)은 같은 교안을 돌려도 결과가 약간 달라질 수 있음
- 자동 감지 항목(형태소 분석, 스펠체크 등)은 매번 동일한 결과

**A-4. PDF 추출 한계**
- PDF 변환 과정에서 띄어쓰기 누락, 줄바꿈 위치 변경 등이 발생할 수 있음
- PDF에서는 띄어쓰기 검수 정확도가 PPT보다 낮음

**A-5. OCR(이미지 판독) 오독**
- OCR로 판독된 텍스트는 글자를 잘못 읽을 수 있음 (예: "를" → "특")
- 따라서 OCR 기반 항목은 어떤 경우에도 "오류 확정"이 아닌 "확인 필요"로만 분류

---

### B. 일부러 안 잡는 것 (오탐 방지를 위한 트레이드오프)

이 항목들은 기술적으로 잡을 수는 있지만, **잡으면 오탐(잘못된 경고)이 너무 많아져서** 의도적으로 제한한 것입니다.

**B-1. 코드/변수명 안의 영문 오탈자 (언더스코어 필터)**
- `my_varable` 같은 변수명 안의 오탈자는 제외 (코드 변수명은 의도적인 명명일 수 있음)

**B-2. 소문자 영단어 스펠체크 제외**
- 소문자로만 된 영단어는 코드/명령어일 가능성이 높아 제외

**B-3. 대문자 약어 제외**
- SQL, DBMS, API 등 대문자 약어는 스펠체크 대상에서 제외

**B-4. camelCase / PascalCase 제외**
- firstName, DataFrame 등 프로그래밍 관례적 표기는 제외

**B-5. 코드 컨텍스트 줄 전체 제외 (영문 스펠)**
- `import`, `def`, `SELECT` 등 코드로 판단되는 줄은 영문 스펠체크 제외

**B-6. 외래어 표기 차이 → 오류가 아닌 "참고사항"**
- 컨텐츠/콘텐츠, 멤버쉽/멤버십 등 실무에서 혼용되는 표기는 오류로 잡지 않고 참고사항으로만 안내

**B-7. 띄어쓰기 — 한글 2자 이하 어절 제외**
- 짧은 단어의 띄어쓰기는 오탐이 많아 제외

**B-8. 띄어쓰기 — 숫자+단위 조합 제외**
- "10분", "3개" 등 숫자+단위 조합은 띄어쓰기 검수 제외

**B-9. 띄어쓰기 — 과도 분리 (4어절 이상) 제외**
- 한 단어를 4어절 이상으로 쪼개는 교정은 오탐 가능성이 높아 제외

**B-10. 반복 변형 — 숫자·구두점 차이 제외**
- "Step 1" vs "Step 2"는 의도적 차이이므로 변형으로 잡지 않음

---

### C. 검수 유형별 신뢰도 요약

| 검수 유형 | 신뢰도 | 비고 |
|-----------|--------|------|
| 한글 오탈자 (형태소 기반) | 높음 | 사전에 없는 단어를 자동 감지 |
| 영문 스펠링 | 높음 | 코드/약어 자동 필터링 |
| 텍스트 깨짐 | 매우 높음 | 패턴 매칭으로 확실한 감지 |
| 반복 변형 | 높음 | 빈도 비교 기반 |
| 띄어쓰기 | 보통 | PDF에서는 정확도 낮음 |
| 문맥 검수 (AI) | 보통 | 매번 결과가 약간 다를 수 있음 |
| OCR (이미지 판독) | 낮음 | 오독 가능성 있어 전부 "확인 필요" |

---

### D. PPT vs PDF 감지율 차이

| 검수 항목 | PPT | PDF |
|-----------|-----|-----|
| 오탈자 | ⭕ 높음 | ⭕ 높음 |
| 영문 스펠링 | ⭕ 높음 | ⭕ 높음 |
| 띄어쓰기 | ⭕ 높음 | ⚠️ 보통 (추출 한계) |
| 텍스트 깨짐 | ⭕ 높음 | ⭕ 높음 |
| 문맥 검수 | ⭕ 높음 | ⭕ 높음 |

**결론: PPT 원본이 있으면 반드시 PPT로 검수하세요.**
